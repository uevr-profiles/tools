## Description

New beta version. I'd need some feedbacks before updating the Nexus page and make an official release. If some of you have the time, would be hugely appreciated. 🍻 

Changelog :
- Weapons can be handled with 2 hands (quick press on left grip to grab).
- Hands animation/finger placements tweaked for each weapons 
- Some right hand animation/finger placements fixed
- Recoil will kick more or less depending if you grab weapons with two hands or not (can be disabled in mod settings).
- Recoil values refined
- The left hand impacting the weapon pivot when two handed can be disabled in the mod settings (Required by someone using a gun stock).
- Fixed hands mesh sometimes not correctly applying the camo/stealth changes
- Cutscenes warp issue fixed, they can now be seen in full 3D for all hmds.
- Several settings added for comfort during cutscenes (Black borders, 2D/3D toggle, smoothing camera movements)
- Fixed the mod menu VR cursor (now fetch the cursor position from UEVR so it'll stay synchronised)
- Fixed Backpack menus and radio UI 3D rendering (Radio is perfect, the others still need some work)
- Fixed Scope zoom view ingame and during cutscenes (cutscenes are fine, ingame will need more work/another way to do it)
- Various UObject lifecycle issues fixed
- Camera being yeeted to the moon in case of ultra low framerate/stutterings fixed.
- Mod menu improved

Known issue :
- The first ingame cutscene after retrieving the backpack from the tree makes the camera going temporarily under the floor
- 3D models are too close and aren't correctly positioned in weapon/items and food menus
- This game needs an input change. Having the crouch and roll inputs on the left hand isn't practical. Will change that at some point.
- It seems UEVR has several rendering issues with SteamVR/Steam link no matter the runtime (OpenVR and OpenXR). I'd like some more feedbacks about that. Virtual Desktop is the way to go here.
- Mod menu renders only in VR, not in the spectator view.
